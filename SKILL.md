# Daily Report

## Inputs to collect

首次使用需确认以下配置（skill 按顺序尝试从环境变量/配置文件读取，找不到则提示用户输入）：

- 邮箱地址
- IMAP 服务器地址（如 imap.example.com）
- IMAP 端口（默认 993）
- 是否使用 SSL（默认 true）
- 邮箱密码或授权码
- 日报存放目录（可选，默认为当前目录下的 Daily Report 文件夹）

## Procedure

1. **建立 IMAP 连接**
   使用用户配置的 IMAP 服务器和端口，SSL 根据配置决定。

2. **确定日期范围**
   - 今日日报：`date.today()` 全天
   - 周五或用户指定「周报」：本周周一 00:00 至周五 23:59:59

3. **拉取邮件**
   用 `imaplib` + `ssl` 连接，搜索 `SINCE <日期>` + `BEFORE <次日>` 过滤。
   对每封邮件提取：发件人、主题、正文（纯文本，忽略 HTML 和附件）。

4. **LLM 归纳（自动推断分类）**
   将邮件列表组织成结构化 prompt，发送给 LLM。

   **输出语言：中文（简体）**

   **Prompt 设计原则**：
   - 不预设任何行业分类或结构
   - 让 LLM 根据邮件内容自动识别：这是什么行业？有哪些工作类型？
   - LLM 自主推断的分类可能包括但不限于：项目管理、客户服务、技术开发、财务、人事、市场销售、行政等
   - 如果邮件内容指向特定行业（如电商、教育、医疗），LLM 应识别并使用该行业的术语

   **日报格式（LLM 自主推断结构，自然语言）**：
   - 根据本日邮件内容自动生成分类标题
   - 每个分类下用自然语言描述，不使用表格
   - 末尾列出「待跟进」或「明日计划」（如适用）

   **周报格式（LLM 自主推断结构，自然语言）**：
   - 根据本周邮件内容自动识别工作主线和分类
   - 按时间顺序或重要性排列
   - 列出 3-5 条下周优先事项

5. **输出报告**
   - 保存为 `YYYY-MM-DD-日报.md` 或 `YYYY-Wxx-周报.md`
   - 直接输出给用户查看

## Output contract

- 本地 Markdown 文件：`{日期}-日报.md` 或 `{年份}-W{周数}-周报.md`
- 报告由 LLM 根据邮件内容自动推断行业和结构，无固定模板

## Failure handling

- IMAP 连接失败 → 检查网络 + 邮箱 IMAP 是否开启 + 服务器地址是否正确
- 无邮件 → 输出「今日/本周无邮件」提示用户
- LLM 归纳失败 → 降级为原始邮件列表（subject + sender + 日期）
- 授权码错误 → 提示用户在邮箱设置中开启 IMAP 并获取授权码
